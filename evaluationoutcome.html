    • Evaluation Outcome

    After the user testing, most of the users believed that our current model is interactive enough, but more sample sentences needs to be added to make our product more responsive.

    Another main improvement is that, we should include a short introduction when the app is installed so that the users know what they can do with our product; and what our product expects them to say. It is definitely a feature that we want to add to our product. It will make it much easier to use.

    Another problem that our users have noticed is that our product always requires the users to say full sentences, which our users find very frustrating. To resolve this issue, we will use the “session feature” of the Alexa development API. It will make Alexa ask for specific information instead of always requiring the users to say the full sentence. The overall impression of the product is satisfactory. They believe our prototype is fully functional, but we should still try to make it more responsive and add more features.

    We have made a list of these issues, categorised by importance (1 the most important, 3 the least) and we have been trying to find a solution to each of them before the actual development process. More issues will be added if we notice some more potential usability problems.

    Issue Importance Solution
    Users not sure what Alexa expects the users to say 1 Include a short introduction / Instruction when the app is first installed.
    Users find it annoying that they have to say full sentences all the time 1 Use the session attribute to make Alexa more human-like and can ask for specific information.
    Want alexa to confirm the request (Our client wants this after seeing the prototype) 1 Create a confirmation before processing the request.
    Add more sample sentences to make Alexa more responsive. 2 We can think of more sentences that the users might say to Alexa and add it to the Alexa skills kit.
